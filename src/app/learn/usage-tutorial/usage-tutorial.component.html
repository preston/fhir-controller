Usage Tutorial

Getting Started with FHIR Controller

FHIR Controller is a simple tool for seeding FHIR servers with data and CQL libraries for any number of predefined scenarios. Developers add their custom FHIR data files into official FHIR Controller images to provide you with one-click FHIR data loading utilities that you can use with your own FHIR servers running on your local computer, behind your corporate firewall, and offline environments. This tutorial will guide you through basic features and workflows.

Loader

The Loader tab lets users load seed data and reset their FHIR servers to a default state. Here you can:

  • Upload default sets of FHIR data files
  • Load CQL logic as FHIR Library resources
  • Set alternate local FHIR server URLs
  • Optionally select a different data loading scenario, if configured

Exploring CQL Libraries

The CQL Libraries tab allows you to:

  • Search for and view CQL libraries currently loaded into the server
  • Execute CQL queries against in the server
  • Perform basic editing of CQL and update the server
  • Trigger running of CQL on the server

Configuration Management

The Configuration tab provides the ability to edit the FHIR Controller application configuration file as currently loaded, temporarily apply it to test your changes, download it, and even create a new configuration from scratch.

Settings

The Settings tab lets you:

  • Configure application preferences
  • Manage user interface options
  • Opt in to experimental and developer settings
Tip: Start with the sample data provided in the Loader tab to get familiar with the interface before loading your own data.
