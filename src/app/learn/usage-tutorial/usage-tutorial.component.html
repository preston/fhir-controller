Getting Started with FHIR Controller

FHIR Controller is a simple tool for seeding FHIR servers with data and CQL libraries for any number of predefined scenarios. Developers add their custom FHIR data files into official FHIR Controller images to provide you with one-click FHIR data loading utilities that you can use with your own FHIR servers running on your local computer, behind your corporate firewall, and offline environments.

This tutorial will guide you through basic features and workflows.

Loader

The Loader tab lets users load seed data and reset their FHIR servers to a default state. Here you can:

  • Upload default sets of FHIR data files
  • Load CQL logic as FHIR Library resources
  • Set alternate local FHIR server URLs
  • Select different data loading scenarios

CQL Libraries

The CQL Libraries tab allows you to:

  • Search and view CQL libraries
  • Execute CQL queries against the server
  • Edit CQL and update the server
  • Trigger CQL execution on the server

Configuration

The Configuration tab provides the ability to edit the FHIR Controller application configuration file as currently loaded, temporarily apply it to test your changes, download it, and even create a new configuration from scratch.

Settings

The Settings tab lets you:

  • Configure application preferences
  • Manage user interface options
  • Opt in to experimental settings

Understanding Scenarios

Scenarios are predefined collections of FHIR resources and CQL libraries that represent specific use cases or testing environments. Each scenario contains a curated set of data files that work together to create a particular testing or demonstration context.

Scenarios are used to:

  • Quickly set up your FHIR server with relevant test data
  • Load subsets of data for isolated use cases without cluttering your server
  • Provide variations of data file combinations for testing
  • Reduce the compute footprint of your FHIR server

When you select a scenario in the Loader tab, FHIR Controller will load all the associated FHIR resources and CQL libraries for that scenario into your configured FHIR server, giving you a complete, ready-to-use data environment.

Tip: Start with the sample data provided in the Loader tab to get familiar with the interface before loading your own data.
